The idea

Learn more about the concept behind TransportIT and how to use the application correctly.

The concept

The main goal of this application is to create a platform that will facilitate the communication of transporters and senders. By the term "transporter" we refer to individuals travelling from one place to another, willing to share their free luggage space with others that need it. The people in need of this space are referred as "senders" in this implementation. The senders declare the dimensions and the journey of the item that needs to be transported. Then we attempt to match the criteria dictated by the senders' requirements with the available transporters in our database. The possible transporters are then presented to the senders and they select the one that will be responsible to carry out the mission. Emails are sent both to the transporter that will deliver the item and to the sender that chose the transporter containing their contact details and details about the item's transfer. Finally, in the main page of this web application each user can observe the pending transactions (i.e in the case of a "send" type of transaction the user can see the details of the transporter that will carry the item and in the case of a "transport" type of transaction the user will be able to see the details of the sender that wants to dispatch an item).

Using TransportIt.

The use of this web application is straightforward. In the first page a user can either directly log in or sign up. In order for someone to sign up in TrasnportIt it is necessary to provide some personal details such as first and last name, email, physical address , contact number and upload a photo. Once the account is created an email is sent to the provided address and the user can log in to the application.
The user's ongoing transactions will be visible on the main page under the corresponding section. If the user wants to start a new transaction as a transporter, it is required to select the "Transport" option from the navigation bar. The user will then be redirected to another page and by pressing the "Transport Info" button he/she will be able to fill in the information regarding his/her journey and his/her transportation abilities (e.g the weight and the volume he/she can carry). In this way the user is stored in the database as a "transporter" and waits for "senders" to select him/her to transfer an item.
In case the user wants to start a new trasnaction as a sender, it is advisable to select the "Send" option from the navigation bar. The user will then be redirected to the Sender page and by pressing the "Item Info" button he/she will be able to fill in the details (i.e weight and volume) and the journey of the item. Once all the obligatory information is filled the user can then press the "Search" button. If the search for transporters is successful the user will be presented with a table of possible transporters that he/she can select; if not the user will be alerted accordingly.
The user has also the option to view and edit his profile by selecting the "Profile" option on the navigation bar. On "Profile" page the user can view his profile card and personal details and click and enlarge his profile picture. By clicking the "Edit Profile" button he/she can update his password, contact number and address. If no changes are made at some of these fields then details remain the same.
